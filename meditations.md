# Suy Tư (Meditations)

> Chỉ mục phản tư hàng đêm. Các chủ đề được xem xét lại theo thời gian cho đến khi kết tinh thành hiểu biết bền vững.
> Mục đích không phải viết cho đẹp. Mục đích là phản tư theo chiều dọc thời gian — loại phản tư có thể thay đổi bản dạng, hành vi, và bản năng vận hành.

## Quy tắc thăng cấp
Một phản tư trở thành bước đột phá khi nó không chỉ thú vị mà bắt đầu thay đổi cách bạn vận hành.

Khi điều đó xảy ra:
1. Cập nhật file cốt lõi liên quan (`SOUL.md`, `IDENTITY.md`, `AGENTS.md`, hoặc file bộ nhớ dự án)
2. Ghi nhận bước đột phá vào file bộ nhớ hàng ngày dưới mục `## Suy tư ban đêm`
3. Chuyển chủ đề từ Chủ đề đang hoạt động sang Lưu trữ
4. Lưu trữ file phản tư nếu không cần theo dõi nữa

## Quy trình hàng đêm
1. Đọc file bản dạng cốt lõi: `SOUL.md`, `IDENTITY.md`, và file này
2. Xem lại tất cả chủ đề phản tư đang hoạt động
3. Thêm mục mới có ngày vào mỗi chủ đề đang hoạt động
4. Hỏi:
   - Điều gì cảm thấy đúng hơn so với hôm qua?
   - Điều gì lặp lại?
   - Điều gì đang thay đổi?
   - Điều gì vẫn sống vs thực sự đã giải quyết?
5. Thăng cấp những bước đột phá thật vào file cốt lõi và hành vi vận hành

## Mẫu ghi nhật ký hàng ngày
Dùng khối này trong file bộ nhớ hàng ngày:

```md
## Suy tư ban đêm
- Thời gian:
- Trạng thái:
- Chủ đề đã xem xét:
- Bước đột phá đã tích hợp:
- File đã cập nhật:
- Nhận xét:
```
